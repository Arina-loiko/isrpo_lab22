# Lab22_JS_Interactive_Loiko

## Описание проекта

Лабораторная работа №22 - разработка интерактивных веб-приложений на JavaScript

## Функциональность калькулятора

### Поддерживаемые операции
- Сложение (+)
- Вычитание (-)
- Умножение (*)
- Деление (/)
- Числа с десятичной точкой

### Ограничения
- Нельзя вводить два оператора подряд
- Нельзя вводить несколько точек в одном числе
- Выражение не может начинаться с оператора (кроме минуса)
- Деление на ноль возвращает ошибку

### Обработка ошибок
- При некорректном выражении на экране появляется "Ошибка"
- После ошибки при нажатии любой кнопки (кроме C) калькулятор очищается
- Кнопка C всегда сбрасывает состояние калькулятора

## Архитектура

```

tic-tac-toe/
  index.html

```

### Класс Calculator

- `constructor(displayId)` - получает ссылку на дисплей, инициализирует переменные
- `init()` - навешивает обработчики на кнопки
- `handleButtonClick(value)` - распределяет нажатия между методами
- `addToExpression(value)` - добавляет символ с валидацией
- `calculate()` - вычисляет выражение через safeEvaluate
- `safeEvaluate(expr)` - изолирует eval, проверяет корректность
- `isValidExpression()` - проверяет на двойные операторы и оператор в конце
- `clear()` - сбрасывает состояние
- `updateDisplay()` - обновляет поле вывода
- `showError()` - переводит в состояние ошибки
- `isOperator(char)` - проверяет, является ли символ оператором
- `isLastCharOperator()` - проверяет последний символ выражения
- `hasDuplicateDot()` - проверяет точку в текущем числе

### Схема взаимодействия

```
index.html -> подключает main.js (тип module)
main.js -> импортирует Calculator из calculator.js
Calculator -> управляет DOM через displayId
Кнопки -> click -> handleButtonClick -> addToExpression / calculate / clear
```

## Логика работы

### Нажатие кнопок

1. Пользователь нажимает кнопку
2. Вызывается handleButtonClick с текстом кнопки
3. Если была ошибка и нажата не C - сначала очищается состояние
4. C - вызывается clear()
5. = - вызывается calculate()
6. Иначе - вызывается addToExpression()

### Валидация ввода

- Перед добавлением символа проверяется корректность
- Оператор после оператора - игнорируется
- Точка в числе где уже есть точка - игнорируется
- Оператор в начале (кроме -) - игнорируется

### Вычисление

1. Проверяется isValidExpression()
2. Проверяется деление на ноль
3. Вызывается safeEvaluate -> eval()
4. Результат записывается в expression и выводится на дисплей
5. При ошибке вызывается showError()
